## 📝 ドキュメント更新ガイドライン

### 更新が必要なタイミング
- 新機能の追加
- アーキテクチャの変更
- 環境変数の追加
- 依存関係の更新
- セキュリティルールの変更

### ドキュメント作成時の注意点
1. **明確な目的**: 誰が何のために読むのかを明確に
2. **具体的な例**: コードサンプルやスクリーンショットを含める
3. **最新性**: 実装と矛盾しないように定期的に更新
4. **検索可能性**: 目次やリンクを適切に配置

### ドキュメントフォーマット
- Markdown形式（`.md`）を使用
- 見出しは階層構造を保つ
- コードブロックには言語指定を追加
- 絵文字を適度に使用して視認性を向上
